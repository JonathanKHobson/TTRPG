    Quick overview

    War Table is a war encounter tool built for fast setup, clear battle outcomes, and quick table reference.

    Battle
    Run engagements with full transparency: pool math, roll details, comparisons, war report, and auto-resolve.
    Armies / Factions
    Manage reusable army records and faction groups, then load them directly into battle panels.
    History / Settings
    Audit outcomes, export/import data, and configure weather, battle rules, and UI behavior.
    Quick Start: run your first battle in under 1 minute
    1. Set Attacker and Defender Army Class.
    2. Enter Army Size and STR for each side.
    3. Leave Dice to commit blank to auto-commit full pool, or enter a number to commit manually.
    4. Click Battle. Review summary, then open Roll Details / Dice Comparisons for full audit.
    Data safety and wipe behavior
    • Reset on Battle tab resets panel runtime values, but does not erase saved armies/factions/history.
    • Wipe all local data in Settings permanently clears saved records, history, and settings on this device.
    • Use Export full state (JSON) before major edits to keep a recoverable backup.

    Battle Guide

    Battle flow

    This section shows how to read each battle result quickly and confidently.

    Commit Dice behavior (single-input model)
    • Each side has one Dice to commit input.
    • Blank input = auto-commit full computed pool.
    • Numeric input = manual commit, clamped to available pool.
    • In compare mode, attacker needs at least 5 committed dice unless cap override is active.
    How to read Dice Math Breakdown
    • Shows pool contributors line-by-line: base, matchup, weather/phenomena, doctrine, nearby assist/hinder, manual delta, cap behavior.
    • Advantage/disadvantage are shown as roll modifiers in breakdown and applied at roll stage.
    • Breakdown is for transparency; it does not introduce hidden modifiers outside displayed lines.
    War Report, Battle Again, and Auto-Resolve
    • War Report opens after Battle when that setting is enabled.
    • Battle Again runs one more round immediately.
    • Auto-Resolve Until Stop loops rounds until a stop condition occurs.
    • Stop reasons include attacker defeated, defender defeated, mutual destruction, phenomena triggered, or safety limit reached.

    Armies & Factions

    Army records and Save vs Update behavior
    • Battle panel button changes label to Update Army when a saved army is loaded on that side.
    • Without a loaded army, button is Save as Army.
    • Armies with size 0 are shown as defeated in Armies list.
    Faction management behavior
    • New Faction modal supports empty factions and assigning existing armies before save.
    • Faction names are unique case-insensitively.
    • Assigning an army in faction flow is authoritative (army moves to the selected faction).

    Settings Reference

    Use this section to translate settings labels into practical behavior.

    Battle Rules settings
    Resolution mode
    Compare-style uses per-comparison resolution. Quick uses total comparison with configured loss preset.
    Quick mode losses
    Controls loser/tie loss values in quick mode only.
    Dice caps
    Enforce normal dice cap behavior or allow over-cap flows where supported.
    Override Doctrine Eligibility
    When enabled, doctrine picker shows all doctrine options regardless of class restrictions.
    Assist/Hinder system
    Master toggle for nearby assist/hinder effects in preview and battle math.
    Assist/Hinder dice cap per side
    Caps net assist/hinder dice delta per side; roll-manip effects are not capped by this number.
    War Report popup after Battle
    Shows/hides post-battle modal presentation. Battle/history still resolve regardless.
    Weather, season, and theme settings
    Season / Climate / Turns per season
    Drives weather randomization pools and turn progression behavior.
    Current turn / Day starts / Night starts
    Controls time-of-day context used by weather-sensitive checks.
    Lock season / Lock weather
    Prevents automated progression for the locked dimension.
    Phenomena base chance
    Base chance used when evaluating random phenomena trigger events.
    Phenomena lock duration / Intensity drift mode
    Controls persistence and intensity evolution once phenomena are active.
    Theme toggles + Theme strength
    Visual layer only. Does not alter battle mechanics.

    War Report & History

    War Report details explained
    • Top banner shows canonical outcome precedence (victory/holds/defeated/mutual).
    • Summary rows show losses and updated sizes for both sides.
    • View Details includes weather used, next weather, roll details, dice comparisons (if compare mode), and math breakdown.
    History card behavior
    • History auto-saves after each battle.
    • Collapsed card = mini war report (headline + losses + updated sizes).
    • Expanded card includes context snapshots, rolls, comparisons, modifiers, and stop reason where present.
    Auto-resolve stop reasons (plain language)
    • Attacker defeated: attacker size reached 0.
    • Defender defeated: defender size reached 0.
    • Mutual destruction: both sizes reached 0 in same resolution.
    • Strange Phenomenon triggered: post-battle weather transition triggered new phenomena for next battle.
    • Safety limit reached: protective max-round stop to avoid infinite loops.

    FAQ / Troubleshooting

    Why did CSV import fail with “No Changes Applied”?

    CSV imports are checked before changes are made. If any row has an error, fix the listed issues and re-import.

    Why was an army updated instead of created?

    Armies CSV matches by army name (not case-sensitive). If the name already exists, that army is updated instead of creating a duplicate.

    Battle button is disabled. What am I missing?

    Common blockers are missing classes/sizes/STR, invalid values, or compare-mode commit rules not satisfied. Check validation message directly above the battle controls.

    Auto-resolve stopped unexpectedly

    Open War Report status line for stop reason. It stops on defeat, newly triggered phenomena, or safety limit guardrail.

    Can I recover wiped data?

    No built-in recovery exists after wipe. Export full state JSON regularly for backups before major changes.

    Glossary

    Compare-style
    Resolution mode that resolves battle through dice comparisons rather than a single total-only check.
    Quick mode
    Simplified total-based resolution with configurable loss presets.
    Commit
    Number of rolled dice from computed pool used in the current round.
    War Report
    Post-battle modal summary with optional deep details and round chaining actions.
    Phenomena lock
    Number of battles a triggered phenomenon remains active before returning to random progression.
    Assist/Hinder
    Nearby army system that applies explicit rules-authored support/pressure effects.
    Command Points (CP)
    Optional campaign-layer resource representing strategic coordination and high-level war leverage.
    Morale Meter
    Optional -5 to +5 campaign pressure tracker for cohesion and combat confidence.
    Supply Meter
    Optional -5 to +5 campaign tracker for logistics, readiness, and attrition pressure.
    Influence Meter
    Optional -5 to +5 tracker for political gravity, alliance pressure, and narrative visibility.

    Rules: Strategic Core

    This is the high-use procedure for resolving strategic battles at the table.

    StepWhat to doWhy it matters
    0Set conditions (weather, time, context)Conditions shape pool math before rolling.
    1Confirm attacker, defender, and engagement setupPrevents mismatched assumptions before commit.
    2Build dice pools from Size/STR and modifiersThis defines each side’s battle leverage.
    3Commit dice to rollCommit choices trade certainty for flexibility.
    4Roll and resolve comparisonsCompares committed power directly and determines losses.
    5Apply losses and decide continue/stopSets immediate outcome and next tactical choice.
    Worked example: pool construction snapshot
    • Attacker Size 150 gives +3 dice; STR 120 gives +12 dice.
    • Defender Size 100 gives +2 dice; STR 80 gives +8 dice.
    • Apply weather/terrain/matchup/doctrine effects, then cap to attacker 10 / defender 8 unless override paths are active.
    • Commit then roll; defender wins ties in compare-style resolution.
    Campaign procedures (movement, aftermath, occupation)
    • Move declarations, retreat handling, and occupation outcomes are usually adjudicated by the GM at campaign scale.
    • Aftermath options include convert, capture, raze, and liberate based on campaign goals and story stakes.

    Rules: Doctrines

    Doctrines are strategic directives. Pick them to shape how an army fights, what it trades away, and what pressure it can sustain.

    Doctrine categories and baseline cost rule
    • Augment doctrines apply to specific armies.
    • Trade doctrines apply as broader strategic toggles.
    • Quiet trade doctrines are stealth/influence-oriented strategic policies.
    • Baseline CP cost is generally defined for size 1-100 and scales by additional 50-size increments.

    Rules: Weather & Terrain

    Weather and terrain are battle context multipliers. Treat them as strategic pressure, not just flavor.

    Condition overview and interpretation
    Time of day overlay
    Day and night alter visibility/light context but are not weather tags by themselves.
    Weather stack
    Weather affects visibility, movement pressure, and matchup conditions in layered ways.

    Core weather tags include clear, cloudy, heavy clouds, fog, rain, heavy rain, wind, thunderstorm, freezing, snow, blizzard, dry, scorching heat, smog/ash haze, and strange phenomena.

    Strange phenomena list (full coverage)
    PhenomenonHow it feelsAt-table effect cue
    AshfallBurned-sky pressure and visibility drag.Plan for longer attrition and reduced clarity.
    Solar EclipseDaylight collapses into omen-dark conditions.Treat many checks as night-like and uncertain.
    Strange LightsLuminous anomalies distort normal cues.Visibility and timing calls can shift unexpectedly.
    Meteor ShowerSky hazard and falling-fire disruption.Add urgency and movement-risk pressure.
    Malevolent StormOppressive storm front with hostile momentum.Expect broad pressure and failed coordination.
    Wild Magic StormArcane instability with surge-level drama.Magic outcomes spike; table events become volatile.
    Magnetic StormDirectional/field interference and pull.Target communication and formation friction.
    Elemental DisturbanceConflicting elemental forces break stability.Resolve element context carefully before commits.
